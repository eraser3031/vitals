# Vitals — 프로젝트 의사

## 한 줄 요약

프로젝트의 생사를 진단하고, 교훈을 남기는 Mac 앱.

## 배경

개발자는 수많은 프로젝트를 시작하지만, 제품화되는 경우는 드물다. 대부분은 방향성 상실, 코드 복잡도 증가, 의욕 저하, 마케팅 부재 등 다양한 이유로 중단된다. 바이브코딩 시대가 오면서 프로젝트를 시작하는 사람은 더 늘어나지만, 프로젝트 관리 경험은 부족한 경우가 많다.

**문제**: 프로젝트가 반복적으로 실패해도, 그 원인을 체계적으로 분석하고 다음에 반영하는 과정이 없다. 그리고 사후분석을 해도 "기억에만 의존한 자기보고"가 되기 쉬워 실제 기록(커밋, 문서)과 어긋난다.

**가설**: 프로젝트에 "의사"가 있다면 — 죽은 프로젝트에서 교훈을 얻고, 위험한 프로젝트를 살려내고, 건강한 프로젝트를 미리 점검할 수 있다면 — 개발자의 프로젝트 성공률이 높아질 것이다.

## 핵심 컨셉: 3가지 진료 축

### 1. 부검 (Autopsy)
> 프로젝트가 이미 죽었을 때 — 사망 원인 해부, 교훈 추출

### 2. 응급 진료 (Emergency)
> 프로젝트가 죽어가고 있을 때 — 위험 신호 분석, 살릴지 접을지 판단

### 3. 검진 (Checkup)
> 프로젝트가 정상적으로 진행 중일 때 — 잠재 리스크 조기 감지

## 현재 제품 방향

사후분석 문서를 **혼자 쓰는 일기**가 아니라 **기록과 대조된 문서**로 만드는 방향으로 좁혀 가고 있다.

- 에디터 안에서 GitHub 커밋 / Notion 페이지를 컨텍스트로 연결
- 작성 중인 문장을 AI가 **팩트체크**하여 기록과 다른 부분을 짚어준다
- 선택한 문장을 우클릭하여 **정교화** — 기록에 맞는 표현으로 제안받고 교체

즉, 사용자의 기억을 AI가 실제 기록으로 교차검증해주는 사후분석 에디터.

## 타겟 사용자

**1차: 인디(1인) 개발자**
- 다수의 프로젝트를 동시에 또는 순차적으로 진행
- 혼자 의사결정하고, 복기할 상대가 없어 교훈이 휘발되는 사람

**2차 (자연 유입): 바이브코딩 입문자**
- AI로 코딩을 시작했지만 프로젝트 관리 경험이 부족한 사람

## 기술 스택

- Electron 33 + React 18 + TypeScript + Vite 5
- Tiptap (에디터) + Tailwind CSS
- GitHub / Notion OAuth (CF Worker 프록시 경유)
- Vercel AI SDK (팩트체크 / 정교화)
- macOS 우선 — 네이티브 코너 반경 모듈 (Objective-C++ N-API)

## 제품 구조

### Electron 앱
- 포스트(사후분석 문서) CRUD
- GitHub/Notion OAuth + 컨텍스트 연결
- 팩트체크 + 정교화(우클릭 팝업)

### Worker (별도 레포)
- GitHub/Notion OAuth 토큰 교환 프록시
- AI API 프록시 (`/ai/chat`, `/ai/refine`)

## 다음 단계
