# Vitals — 프로젝트 의사

## 한 줄 요약

프로젝트의 생사를 진단하고, 교훈을 남기는 Mac 앱.

## 배경

개발자는 수많은 프로젝트를 시작하지만, 제품화되는 경우는 드물다. 대부분은 방향성 상실, 코드 복잡도 증가, 의욕 저하, 마케팅 부재 등 다양한 이유로 중단된다. 바이브코딩 시대가 오면서 프로젝트를 시작하는 사람은 더 늘어나지만, 프로젝트 관리 경험은 부족한 경우가 많다.

**문제**: 프로젝트가 반복적으로 실패해도, 그 원인을 체계적으로 분석하고 다음에 반영하는 과정이 없다.

**가설**: 프로젝트에 "의사"가 있다면 — 죽은 프로젝트에서 교훈을 얻고, 위험한 프로젝트를 살려내고, 건강한 프로젝트를 미리 점검할 수 있다면 — 개발자의 프로젝트 성공률이 높아질 것이다.

## 핵심 컨셉: 3가지 진료 축

### 1. 부검 (Autopsy)

> 프로젝트가 이미 죽었을 때

- 사후 분석을 통해 사망 원인을 파악한다
- 타임라인, 의사결정, 전환점을 복기한다
- 교훈을 구조화하여 다음 프로젝트에 반영할 수 있게 한다

### 2. 응급 진료 (Emergency)

> 프로젝트가 죽어가고 있을 때

- 현재 프로젝트의 위험 신호를 분석한다
- 과거 부검 데이터를 기반으로 유사 패턴을 진단한다
- 살릴 수 있는 방법을 제시한다

### 3. 검진 (Checkup)

> 프로젝트가 정상적으로 진행 중일 때

- 정기적으로 프로젝트 건강 상태를 점검한다
- 잠재적 위험 요소를 조기에 발견한다
- 건강한 상태를 유지하기 위한 권장 사항을 제공한다

## 기술 스택

- **프레임워크**: Electron
- **프론트엔드**: React + TypeScript
- **빌드**: Vite
- **플랫폼**: macOS (우선), 크로스 플랫폼 확장 가능

## 타겟 사용자

**1차: 인디(1인) 개발자**

- 다수의 프로젝트를 동시에 또는 순차적으로 진행하는 개발자
- 혼자 의사결정하고, 복기할 상대가 없어 교훈이 휘발되는 사람
- 프로젝트 실패를 반복하지만 체계적으로 분석할 도구가 없는 사람

**2차 (자연 유입): 바이브코딩 입문자**

- AI로 코딩을 시작했지만 프로젝트 관리 경험이 부족한 사람
- 1차 타겟이 쓰는 도구를 따라 유입되는 흐름

## 비즈니스 모델

**프리미엄 (무료 + 구독)**

| | 무료 | 유료 (구독) |
|---|---|---|
| 진단 | Claude Code 스킬(터미널)에서 수행 | 앱 내에서 AI 진단 직접 수행 |
| 보고서 관리 | 앱에서 임포트/열람 | 앱에서 임포트/열람 |
| AI 비용 | 사용자 Claude Code 사용량 | 구독료에 포함 |
| 사용 경험 | 터미널 ↔ 앱 이동 | 앱 하나에서 완결 |

**전환 흐름**: 무료로 스킬을 써서 가치 체감 → 보고서 관리 위해 앱 설치 → 편의성 위해 유료 전환

**앱 단 부가 가치 (무료/유료 공통)**: 보고서 간 패턴 분석, 히스토리 비교, 프로젝트별 통계 등 — 스킬만으로는 할 수 없는 축적 기반 인사이트

## 제품 구조

### Claude Code 스킬 (진단 엔진)

- 로컬 프로젝트/GitHub 레포 정보를 기반으로 질의응답 진행
- 부검/응급/검진 3가지 모드
- 진단 완료 후 마크다운 보고서 생성
- 상세 명세: [SKILL_SPEC.md](./SKILL_SPEC.md)

### Electron 앱 (보고서 관리 + 유료 진단)

- 스킬이 생성한 보고서 임포트 및 관리
- 프로젝트별 보고서 히스토리, 패턴 분석
- (유료) 앱 내 AI 진단 — 스킬과 동일한 흐름을 앱 UI로 제공

## MVP 범위

**부검(Autopsy)부터 시작한다.**

부검에서 축적된 사망 원인, 패턴, 교훈 데이터가 응급/검진의 기반이 된다.

- Phase 1: 부검 — 프로젝트 사후 분석 및 교훈 구조화
- Phase 2: 응급 — 부검 패턴 기반 위험 감지 및 대응
- Phase 3: 검진 — 패턴 기반 정기 점검 및 조기 경고
